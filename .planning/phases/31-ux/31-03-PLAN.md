**체크리스트 A — dark 테마 잔재 (CONSIST-03):**

- `bg-.*-900/30` → `bg-*-50`로 교체 (예: `bg-red-900/30` → `bg-red-50`)
- `border-.*-500/30` → `border-*-200`로 교체
- `border-.*-800` → `border-*-200`로 교체
- `text-red-300` → `text-red-600`
- `text-red-200` → `text-red-600`
- `text-red-400` (결과 섹션 에러 안내용) → `text-red-600`
- `text-yellow-400` (안내 박스 텍스트) → `text-yellow-700`
- `text-green-400` (discount 표시는 제외 — 시각적 구분 목적이므로 유지)

**체크리스트 B — 에러 박스 통일 (CONSIST-03):**
에러 메시지는 반드시 박스 형태로:
```jsx
{error && (
  <div className="mb-4 p-3 rounded-lg bg-red-50 border border-red-200">
    <p className="text-sm text-red-600">{error}</p>
  </div>
)}
```
단순 `<p className="text-red-500 text-sm mb-3">{error}</p>` 패턴도 허용 (계산기별 최초 적용 방식 유지). 박스와 p 태그가 혼재하는 경우만 통일.

**체크리스트 C — 경고 메시지 통일 (CONSIST-03):**
경고는 `<p className="text-orange-500 text-sm mb-3">{warning}</p>` 패턴으로 통일.
(경고는 심각도 낮으므로 p 태그 직접 표시 허용)

**체크리스트 D — 숫자 입력 필드 (CONSIST-01, INPUT-04):**
각 계산기의 숫자 금액 필드 확인:
- `type="text"` + `inputMode="numeric"` (원화 금액)
- `type="text"` + `inputMode="decimal"` (소수점 이율, BAC)
- `type="number"` 잔재 없는지 확인 — 있으면 수정

**체크리스트 E — 필수 * 표시 일관성 (FLOW-03):**
7개 계산기 모두 진정한 필수 필드(없으면 계산 불가)에 * 표시 확인. 선택 필드는 "(선택)" 또는 설명 힌트로 구분.

**체크리스트 F — FLOW-01 재검증:**
7개 계산기 모두 초기 상태 `result === null` 조건부 렌더링 확인.
`result !== null && (...)` 패턴이 없으면 추가.

**체크리스트 G — RESULT-02 재검증:**

정수 금액을 그대로 출력(`{n}원`)하는 경우 수정.
