**체크리스트 (파일당 적용):**

1. **CONSIST-03: 에러 스타일 통일**
   - 에러 메시지: `className="text-red-500 text-sm ..."` — 이 패턴이어야 함
   - 경고 메시지: `className="text-orange-500 text-sm ..."` — 이 패턴이어야 함
   - 결과 섹션 info 안내: `bg-blue-50 border border-blue-200` + `text-blue-600` — 이 패턴이어야 함

   - 결과 섹션 danger: `bg-red-50 border border-red-200` + `text-red-700` — 이 패턴이어야 함
   - **금지 패턴:** `bg-*-900`, `bg-*-800`, `bg-*-700`, `text-*-300`, `text-*-400` (경고/에러에 쓰인 경우) — 발견 즉시 수정

2. **INPUT-04: type=number 잔존 확인**
   - `type="number"` 속성이 남아있는 입력 필드를 찾아 `type="text"` + `inputMode="decimal"` 또는 `inputMode="numeric"`으로 변환
   - date 타입 (`type="date"`)은 그대로 유지 (날짜 picker 필요)
   - select/checkbox/radio는 해당 없음

3. **CONSIST-01: 동일 개념 입력 방식 통일**
   - 금액 필드: `type="text"` + `inputMode="numeric"` + `onChange e.target.value.replace(/[^0-9]/g, '')` 패턴
   - 소수 포함 비율/계수 필드: `type="text"` + `inputMode="decimal"` + `onChange e.target.value.replace(/[^0-9.]/g, '')` 패턴

4. **CONSIST-02: 결과 표시 형식 통일**
   - 금액: `formatNumber(n)원` 형식 (천단위 콤마 + 원 단위)
   - 퍼센트: `n%` 형식
   - formatNumber 함수가 없고 `toLocaleString('ko-KR')` 직접 사용하는 경우 → 일관성 있으면 OK

5. **FLOW-03: 필수 필드 * 표시 누락 확인**
   - 계산하기/확인하기/미리보기 실행에 필수인 필드는 라벨에 ` *` 포함
   - 선택적 필드에 * 없어야 함
