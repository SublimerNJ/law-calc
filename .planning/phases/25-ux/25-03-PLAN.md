6개 계산기를 모두 읽고 다음 3가지 CONSIST 요구사항을 기준으로 차이를 파악한 뒤 즉시 수정 (per D-03):

**CONSIST-01: 동일 개념의 입력 방식 통일**
각 계산기에서 금액 입력 방식 확인:
- `handleNumberInput` 또는 동등한 onChange 패턴이 모든 금액 필드에 적용됐는지 확인
- `displayValue` (천단위 콤마 표시) 함수가 모든 금액 input의 `value` prop에 적용됐는지 확인
- `inputMode="numeric"` 속성이 모든 금액 필드에 있는지 확인
- 불일관 발견 시 즉시 통일

**CONSIST-02: 결과 표시 형식 통일**
각 계산기의 결과 섹션 확인:
- 금액 결과에 `toLocaleString('ko-KR')` 또는 `formatNumber()` 적용 여부
- 금액을 "원" 단위로 표시하는지 또는 "만원"/"억원" 단위로 표시하는지 — 혼용이면 통일 기준 결정:
  - 결과 금액이 원 단위 숫자이면 `toLocaleString('ko-KR')` + "원" 텍스트
  - 요약/레이블에서 "억원" 표현은 허용 (가독성)
- 불일관 발견 시 수정

**CONSIST-03: 에러 메시지 스타일·톤 통일**
각 계산기의 에러/경고 메시지 확인:
- 에러 메시지 위치: 계산 버튼 위 (`{error && <p ...>}`) — 버튼 아래에 있으면 버튼 위로 이동
- 에러 스타일: `className="text-red-500 text-sm mb-3"` 형태로 통일
- 경고 스타일: `className="text-orange-500 text-sm mb-3"` 형태로 통일
- 메시지 톤: "~해주세요", "~해야 합니다", "~을 확인해주세요" — 존댓말 일관성 확인
- `error` state 초기화: 새로운 계산 시도 시 이전 에러 제거 — `setError(null)` 또는 `setError('')`를 handleCalculate 시작부에 추가됐는지 확인

**수정 시 주의:**
- 각 계산기의 로직(계산 함수)은 변경하지 않음
- UI/UX 표시 방식만 수정
- inheritance-order는 금액 필드 없음 — 해당 없는 항목 skip
