### CONSIST-01: 동일 개념 입력 방식 통일

**금액 입력 필드 패턴 기준:**
```tsx
type="text"
inputMode="numeric"
value={field ? parseInt(field).toLocaleString('ko-KR') : ''}
onChange={e => setField(e.target.value.replace(/[^0-9]/g, ''))}
placeholder="예: 300,000,000"
className="w-full bg-white border border-slate-200 rounded-lg px-4 py-3 text-slate-900 focus:border-blue-600 focus:outline-none"
```

확인사항:
- 5개 파일 모두 금액 필드에 `type="text" inputMode="numeric"` 사용 여부
- `placeholder` 형식이 "예: 숫자,숫자,숫자" 형식인지 (단위 명시)
- 편차 발견 시: placeholder를 "예: 300,000,000" 형식으로 통일

**날짜 입력 (capital-gains-tax만 해당):**
- `type="date"` — 표준 패턴, 변경 불필요

### CONSIST-02: 결과 표시 형식 통일

**금액 표시 패턴:**
- `formatNumber(n)` = `n.toLocaleString('ko-KR')` 사용 여부
- 5개 파일 모두 `formatNumber` 함수 정의 또는 동일 로직 인라인 사용 여부 확인
- 금액 뒤 "원" 단위 표시 일관성 확인

**비율 표시:**
- 세율 표시: `(rate * 100).toFixed(1)%` 또는 `%` 형식
- comprehensive-income-tax는 `result.rate`가 이미 "6%", "15%" 형식 문자열 — 통일 불필요

**결과 합계 강조:**
- 최종 납부세액(합계)은 `text-xl font-bold` 또는 `text-xl font-semibold` + category.color 사용 여부
- 5개 파일 간 편차 확인. 예: registration-tax에서 등록면허세가 `text-2xl font-bold`인데 합계가 `text-lg font-bold` — 혼재 시 합계만 가장 크게 수정

### CONSIST-03: 에러 메시지 스타일 및 어조 통일

**스타일 기준 (per D-01, D-02):**
- 에러 (Critical/Warning): `<p className="text-red-500 text-sm mb-3">{error}</p>`
- 경고 (Info): `<p className="text-orange-500 text-sm mb-3">{warning}</p>`
- 에러 표시 위치: 계산하기 버튼 바로 위

**어조 기준:**
- 필수 입력 누락: "~을 입력해주세요."
- 범위 초과 경고: "~이 N을 초과합니다. 확인해주세요."
- 날짜 역전: "~일이 ~일보다 앞서거나 같을 수 없습니다."
- 결과 0/음수 안내: info 박스 (text-blue-600, bg-blue-50)

5개 파일 각각에서 위 어조에서 벗어난 메시지 발견 시 수정.

- 체크박스, 라디오 버튼, 선택적 필드에는 `*` 불필요
- `*` 앞에 공백 없이 바로 붙임: `"취득가액 (원) *"` 형식

5개 파일 각각에서 누락된 `*` 또는 잘못된 위치에 `*` 있으면 수정.
