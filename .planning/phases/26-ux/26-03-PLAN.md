- `type="text"` + `inputMode="numeric"` + onChange에서 `[^0-9]` 제거
- 표시값: `value ? parseInt(value).toLocaleString('ko-KR') : ''`
- 이 패턴에서 벗어나는 계산기 있으면 수정.

시간 입력 필드 패턴 표준:
- 정수 시간: `type="text"` + `inputMode="numeric"` + `[^0-9]` 제거
- 소수 시간(연장근로): `type="text"` + `inputMode="decimal"` + `[^0-9.]` 제거
- `type="number"` 사용 중인 시간 필드 있으면 교체.

날짜 입력 필드: `type="date"` 그대로 유지 (표준 패턴).

**CONSIST-02: 결과 표시 형식 통일**

금액 결과 표준: `formatNumber(amount)` 또는 `amount.toLocaleString('ko-KR')` — 어느 쪽이든 통일. 각 계산기에 `formatNumber()` 함수가 모두 있으므로 `formatNumber()` 사용으로 통일. 결과 섹션에서 raw 숫자를 직접 `{amount}` 표시하는 경우 수정.

시간 결과 표준: `weeklyHolidayHours.toFixed(1)` 또는 정수 표시 — weekly-holiday-pay 결과에서 시간을 소수로 표시하는 경우 `.toFixed(2)` 또는 `.toFixed(1)` 통일.

